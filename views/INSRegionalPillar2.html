This view will be placed under REGIONAL menu:

Here we have an html table with the capability to change the field that is being used in the table with 3 radio buttons (see Pillar 1 - Standard first). We are creating the table with the "createCube" method. In this Cube, the Dimesions should be:
"=If($(v.SelectDim)=1,[COUNTRY],If($(v.SelectDim)=2,[Global Brand],If($(v.SelectDim)=3,[Global Product])))" : dimension showed depending on the v.SelectDim variable.
"=If($(v.SelectDim)=1,lower([COUNTRY]),If($(v.SelectDim)=2,[%HIDE_BrandPNG]))" : name of the corresponding country flags and brand logos without the .png extension. They change depending on the v.SelectDim variable.
Measures should be taken from the table in Pillar 2 view in the app.

For creating the visualization of the table in the view, a function to change the name of the first column depending on the radio button should be created in the controller.
In the html table, if the user clicks on some dimension value, this value should be filtered in the dashboard. To do that, another function should be created in the controller.
Also if the text of the cell exceeds the cell's size there will appear three dots that, when hover over, will detail the full name. In order to do that we are using "Title" which is an html element and an css class.

For this chart we will use the following expression to obtain the information with the info button:
"maxstring({<[%HIDE_Report_ID]={14} ,[%HIDE_Graph_ID]={73}>}Text)"
