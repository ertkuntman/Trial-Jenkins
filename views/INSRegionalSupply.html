PILLAR 1 - STANDARD

This view will be placed under REGIONAL - Pillar 1 menu:

In this view we have the capability to change the rolling months available that we want in both graphs. In order to achieve this we are using the "getContent" and "setContent" method (which is a call to the qlik sense api) and combining it with "ng-click" which is a directive that will run a function defined in the controller. In those functions we should take the v.month variable and change its content between 1, 2, 3 and 4 values that match with Isolated month, Rolling 3 months, Rolling 6 months and Rolling 12 monts respectively. You can also change the Rolling months buttons class depending on whether they are selected or not with the "ng-class" directive.

Above the pivot table, there are available two radio buttons (Per month and Aggregated). With "ng-init", "ng-click" and "ng-if" directives, both tables available in "Regional - Pillar 1 - Standard" view in the app can be switched between them.

In the combo chart, we have the checkboxes that will control the measures that we have in the Adherence parameteres graph. For those we are using in the view "ng-checked" that will control what is checked and what is not, "ng-model" which will link a variable from the controler with the html and "ng-click" that will control which measure is going to be shown in the chart. In the controller, an object with all the possible measures should be done to include them into the chart created with the api visualization. Whenever you check or uncheck a value of the checkbox we are saving that object in the controller of the view that will have the measures that we will be adding into the chart with the "create combochart" method.
To create the qlik combo chart, the "Dev-tool" extension is quite useful.
In addition, as this chart is not being taken with the get-object method, to download it, we would need to create a function in the controller to download the object by the id. This id will be the one of the object in the app, and all the measures will be always downloaded.

For those two charts we will use the following expressions to obtain the information with the info buttons:
"maxstring({<[%HIDE_Report_ID]={11} ,[%HIDE_Graph_ID]={52}>}Text)"
"maxstring({<[%HIDE_Report_ID]={11} ,[%HIDE_Graph_ID]={54}>}Text)"
